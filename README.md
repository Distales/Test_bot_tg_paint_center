# AI Telegram Assistant для компании «Центр Красок #1»

Telegram-бот с AI-ассистентом, который отвечает на вопросы о компании на основе заранее собранной информации.

## Описание проекта

Этот проект реализует Telegram-бота, который работает как обычный AI-чат без команд и меню. Пользователь пишет обычные вопросы о компании, а бот отвечает только на основе предоставленной информации о компании «Центр Красок #1».

Бот использует AI-модель для генерации ответов, но строго ограничивается предоставленной базой знаний для предотвращения галлюцинаций.

## Стек технологий

- **Python 3.11+**
- **aiogram 3.x** - библиотека для создания Telegram ботов
- **Grok (xAI) API** - для генерации ответов
- **python-dotenv** - для управления переменными окружения
- **requests** - для HTTP запросов к Grok API
- **SQLite** (опционально) - для хранения истории диалогов (в MVP используется in-memory хранилище)

## Как создать Telegram-бота через BotFather

1. Откройте Telegram и найдите @BotFather
2. Отправьте команду `/newbot`
3. Следуйте инструкциям, чтобы создать бота и получить токен
4. Скопируйте полученный токен - он понадобится для настройки

## Как получить API ключ для Grok (xAI)

1. Зарегистрируйтесь на https://console.x.ai/ (или получите доступ через xAI)
2. Получите API ключ из консоли разработчика
3. Скопировать полученный ключ

## Как запустить локально

1. Скопируйте `.env.example` в `.env`:
   ```bash
   cp .env.example .env
   ```

2. Откройте файл `.env` и вставьте ваши токены:
   ```
   TELEGRAM_BOT_TOKEN=your_actual_telegram_bot_token
   GROK_API_KEY=your_actual_grok_api_key
   GROK_MODEL=grok-beta  # или другую модель от xAI
   MAX_CONTEXT_MESSAGES=5
   ```

3. Запустите бота:
   ```bash
   python -m app.main
   ```

## Как запустить через Docker

1. Убедитесь, что у вас установлен Docker и Docker Compose

2. Скопируйте `.env.example` в `.env` и заполните токенами (как описано выше)

3. Запустите контейнеры:
   ```bash
   docker-compose up --build
   ```

## Как обновлять базу знаний

База знаний хранится в двух форматах для удобства:
1. `data/company_knowledge.json` - основной формат, используемый ботом для загрузки
2. `data/company_knowledge.md` - копия в формате Markdown для удобного чтения и редактирования

Для обновления информации о компании:
1. Отредактируйте файл `data/company_knowledge.json` (рекомендуется) или `data/company_knowledge.md`
2. Если редактируете Markdown-файл, скопируйте его содержимое в JSON-файл в поле "knowledge_base"
3. Сохраните файл
4. Перезапустите бот, чтобы изменения вступили в силу

## Ограничения MVP

1. Бот отвечает только на основе предоставленной базы знаний - он не имеет доступа к внешней информации в реальном времени
2. История диалогов хранится в оперативной памяти (in-memory) и будет потеряна при перезапуске бота
3. Нет интеграции с системами заказа или оплаты
4. Бот не может оформлять заказы или проверять реальное наличие товаров
5. Для работы требуются действительные токены Telegram Bot и Grok API

## Как проверить, что бот работает правильно

1. Запустите бота одним из методов выше
2. Найдите вашего бота в Telegram по имени
3. Начните диалог, отправив `/start` или простое приветствие
4. Задайте вопрос о компании, например: "Чем занимается компания?"
5. Убедитесь, что ответ основан только на предоставленной информации и не содержит выдуманных фактов
6. Попробуйте задать вопрос не related к компании - бот должен вежливо отказаться отвечать
7. Попробуйте спросить о ценах или наличии - бот должен дать осторожный ответ с рекомендацией проверить сайт

## Технические детали

### Защита от галлюцинаций

1. Системный промпт явно запрещает выдумывать факты и требует отвечать только на основе базы знаний
2. Перед отправкой вопроса в AI API выполняется проверка на релевантность компании
3. Для определенных типов вопросов (цены, наличие, вакансии, акции) используются специальные шаблонные ответы
4. Если в базе знаний нет информации, бот честно сообщает об этом и предлагает обратиться к менеджерам

### Контекст диалога

Бот хранит последние `MAX_CONTEXT_MESSAGES` (по умолчанию 5) сообщений для каждого пользователя, чтобы поддерживать краткий контекст диалога. Это позволяет пользователю задавать уточняющие вопросы без повторения полного контекста.

### Обработка ошибок

Бот корректно обрабатывает various ошибки:
- Ошибки API Grok
- Ошибки загрузки базы знаний
- Пустые сообщения
- Внутренние ошибки обработки

В случае ошибок бот возвращает понятное пользовательское сообщение и логирует детали для отладки.
